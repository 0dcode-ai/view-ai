# 现在项目怎么做和怎么演示

这份文档记录我们当前定下来的产品路线、项目结构和演示方式。核心结论是：Web 端保留并继续作为主要产品调试台；移动端先做成好看的纯前端演示版，方便拿给别人看；后端独立能力保留，等产品成熟后再逐步接入真实数据。

## 当前判断

项目现在同时存在 Web 端、移动端和独立后端。

```text
view_agent
├── app/                 # 原 Next.js Web 端
├── components/          # 原 Web 组件
├── prisma/              # 原 Web/SQLite 数据模型
├── apps/
│   ├── api/             # NestJS 独立后端
│   └── mobile/          # Expo React Native 移动端
└── packages/
    └── shared/          # 共享 DTO、Zod schema、API client
```

Web 端目前仍然是原来的 Next.js 单体形态，适合继续快速验证功能。移动端已经升级到 Expo SDK 55，当前定位改成“可演示的纯前端 App”，不用后端也能直接打开。

## 产品路线

我们不急着把移动端做成完整产品。更稳的路线是：

1. Web 端继续做主开发和产品调试。
2. 在 Web 端把题库、简历、JD、模拟面试、复盘、冲刺计划、面经、公司情报这些核心功能打磨成熟。
3. 独立后端 `apps/api` 继续保留，后续逐步把 Web 端也切到 `/v1/*` API。
4. 移动端先做精美演示版，覆盖完整功能感知，但数据先内置在前端。
5. 等产品逻辑、数据模型和接口稳定后，再把移动端逐步接入真实 API。

最终理想形态：

```text
Web 端：产品调试台 + 管理后台
Mobile：正式用户端
NestJS API：唯一业务后端
Postgres/Supabase：统一数据源
packages/shared：统一类型和接口契约
```

## 移动端当前定位

移动端现在优先服务演示，不依赖后端、不依赖登录，也不需要 Docker。打开后能直接看到一个完整的面试训练产品样子。

已经做成纯前端演示的模块：

- 今日任务：首页完成度、任务打卡、AI 复盘摘要。
- 题库复习：搜索、分类筛选、展开答案、标记掌握。
- 模拟面试：内置面试题、输入回答、即时评分、改进建议。
- 冲刺计划：JD 匹配度、能力雷达、7 天路线。
- 演示中心：简历解析、JD 匹配、面经情报、实验室、公司情报、复盘卡。

这部分主要代码在：

```text
apps/mobile/App.tsx
```

移动端当前版本：

```text
Expo SDK 55
expo ~55.0.17
react-native 0.83.6
react 19.2.0
```

## 怎么启动移动端演示

演示移动端时，只需要启动 Expo：

```bash
cd /Users/william/coding/projects/view_agent
npx expo start -c
```

然后安卓手机打开 Expo Go，扫终端二维码。

注意：

- 手机里的 Expo Go 需要是 SDK 55 兼容版本。
- 现在移动端是纯前端演示，不需要先启动后端。
- 如果手机显示旧页面，先关掉 Expo Go 里的项目，再重新扫码。
- 如果终端提示端口占用，先停掉旧的 Expo 进程，再重新跑 `npx expo start -c`。

## 怎么启动 Web 端

Web 端仍然是原来的 Next.js 项目，可以继续作为产品调试台：

```bash
cd /Users/william/coding/projects/view_agent
```


Web 端目前还是走原有 Next API routes 和 SQLite。这个阶段它可以继续作为主开发入口。

## 怎么启动独立后端

如果要调试新后端 `apps/api`，先启动本地 Postgres：

```bash
cd /Users/william/coding/projects/view_agent
docker compose up -d postgres
npm run prisma:push -w apps/api
npm run migrate:legacy -w apps/api
npm run dev:api
```

API 文档地址：

```text
http://localhost:4000/docs
```

健康检查：

## 之前遇到的问题和处理

安卓扫码没反应：

- 原因通常是手机访问不到电脑 IP，或者 Expo Go / SDK 版本不匹配。
- 我们把移动端升级到了 Expo SDK 55，和你的 Expo Go 对齐。

手机页面空白：

- 之前是移动端依赖后端数据，后端 `/v1/daily` 一度返回 500。
- 后端 500 的原因是开发用户 ID 和导入用户 email 不一致。
- 现在移动端已经改成纯前端演示版，不再因为后端没开而白屏。

Prisma generated client 找不到：

- 原因是 `nest build` 不会自动把 `src/generated/prisma` 复制到 `dist/src/generated/prisma`。
- 已经通过 Nest assets 和复制脚本修复。

## 下一步建议

短期：

1. 继续美化移动端演示版，让它像一个完整产品。
2. Web 端继续承载真实功能调试。
3. 移动端先不急着接后端，除非某个演示链路必须真实数据。

中期：

1. Web 端功能稳定后，把业务能力逐步沉到 `apps/api`。
2. Web 端也改成调用 `/v1/*`，减少 Next API routes 的历史包袱。
3. 移动端按模块接入真实 API，从题库、今日任务、模拟面试开始。

长期：

1. Supabase Auth + Postgres 成为统一登录和数据源。
2. Web 端成为管理后台和产品实验台。
3. 移动端成为正式用户端。
