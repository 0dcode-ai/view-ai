## 当前产品主线

项目已经从早期“记录 / 面试 / 技术文章 / 创业想法”的功能集合，演进为围绕“一个具体求职机会”的面试准备 SaaS。当前主线是：

1. 今日：承接每日行动、待复习、复盘卡、冲刺任务。
2. 求职机会：对标 Teal，作为简历、JD、匹配、模拟、复盘和下一步动作的核心容器。
3. 面试官 Agent：模拟真实面试官控场、追问、纪要和复盘。
4. 模拟练习：面试者视角，基于简历、JD、候选人准备 Agent 和文稿进行练习。
5. 知识库：承接八股、技术文章拆卡、复习和复盘回流。
6. 开源趋势：发现 GitHub 潜力项目，沉淀为来源材料和技术素材。
7. Agent 设置：统一管理独立 Agent 的配置、运行和日志。
8. 创业想法：保留为产品孵化和演示入口。

其他能力继续作为辅助入口保留，例如技术文章库、简历库、JD 解析、复盘、冲刺计划、趋势、公司情报、面经库、实验室等。

## 2026-05-08：Teal 对标版求职机会工作台

### 已完成

- 新增 `ApplicationActivity`，记录创建机会、状态变化、匹配刷新、简历版本、Agent 运行等关键事件。
- 新增 `ResumeVersion`，支持基于原始简历创建应用内定制版本，不覆盖原简历。

- 支持 `Auto-Select`，按 JD 对简历内容块进行启用、停用和重排。
- 支持针对 missing keyword 生成候选 bullet，先进入候选区，用户确认后再使用。
- 前端求职机会页已具备状态漏斗、机会表格、筛选排序、inline 修改和右侧详情工作台。
- 详情页包含：
  - 概览

  - 匹配
  - 简历版本
  - 面试准备
  - 活动

## 2026-05-07：记录模块持续优化

### 背景

用户经常不是只看到一条孤立八股，而是看到一整篇技术文章、一段 Q&A 或一组面试资料。旧的“新增记录”只能把内容整理成一张卡，容易把多个问题挤在同一张卡里，后续复习和检索都不够清晰。

### 已完成

- “新增记录”弹窗新增两种模式：
  - 单张整理：保留原来的单卡 Agent 能力。
  - 文章拆卡：把整篇技术文章或 Q&A 文本拆成多张八股卡。
- 文章拆卡支持两种输入：
  - 直接粘贴文章正文。
  - 从本地“技术文章”库选择一篇文章，再可选补充额外正文。
- 生成结果先进入多卡预览，不会静默保存。
- 预览里支持逐张勾选、编辑、删除，再批量保存到现有题库。
- 每张草稿包含题目、答案、主题、标签、题型、能力维度、难度、掌握建议、优先级和面试提示。
- 面试提示统一包含：
  - 必说点
  - 常见追问
  - 项目连接
- fallback 模式已可用：没有模型 API key 时，会按明显问题句、标题行和 Q&A 段落本地拆卡。

### 后续升级说明

2026-05-08 已在此基础上升级为“题解式学习稿”。历史的“面试提示 / 必说点 / 常见追问 / 项目连接”仍能兼容解析，但新生成内容会使用更完整的结构：一句话结论、面试回答、考点定位、思路、步骤、示例、复杂度与取舍、易错点、常见追问、项目连接和复习清单。

### 当前数据与接口

- 继续复用 `KnowledgeCard`，不新增记录表。
- Next API：
  - `POST /api/knowledge/agent/batch`：文章转多张八股卡草稿，支持 `rawText` 和 `articleId`。
  - `POST /api/knowledge/bulk`：批量创建知识卡。
- `packages/shared` 已新增批量创建和批量 Agent 输入 schema。
- `apps/api` 已同步批量保存和 rawText fallback 拆卡接口；由于独立 API 当前还没有本地 `TechnicalArticle` 表，`articleId` 选择文章仍由 Web 端 Next API 承接。

## 2026-05-07：面试功能持续优化

### 背景

最早的“面试官”能力更像一份文字版面试文稿：生成问题、追问、信号和评分口径，适合准备题单，但不够贴近真实面试官使用过程。

在继续打磨后，面试功能已经分成两条更清晰的线：

- 面试官工作台：一个 Agent 负责开场、控场、追问、纪要和复盘。
- 面试者练习：围绕题目、结构回答、追问训练和弱点复练持续训练。

### 当前主线

面试功能聚合“面试官”和“面试者”两个视角，用户可以随时切换：

- 面试官：贴简历，可选 JD，选择时长和级别，生成整场固定题纲；过程里自由切题、记录纪要、挂追问和新建自由讨论卡；结束后输出聚合复盘。
- 面试者：基于文稿或题目逐题练习，也可以启动 AI 文字模拟和本地诊断。
- 市场标杆补齐雷达：对照 LeetCode、HackerRank、CodeSignal、CoderPad、interviewing.io、Big Interview、Pramp 等平台公开能力，持续检查我们在个性化模拟、结构化评分、题库沉淀、答案优化、coding 环境和真人压力模拟上的覆盖程度。

### 市场调研沉淀

本轮调研把主流面试开放平台的好能力收敛成 6 个补齐方向：

- JD/简历自适应模拟：优秀平台会让模拟和候选人资料、岗位要求、回答过程联动，而不是只给固定题单。
- 结构化评分卡：企业评测平台强调统一 rubric、维度分、逐题反馈和面试官可复盘材料。
- 公司/主题题库沉淀：高频题、公司标签、主题标签、难度和学习路径是备考平台的底座。
- 答案对照与下一版：用户原回答、扣分点、参考好答案和下一版表达需要放在一起。
- 真实 Coding / 系统设计环境：在线 IDE、协作编辑、数据库/前端任务和系统设计白板是技术面试平台的重要差异点。
- 真人/同伴压力模拟：匿名真人 mock、同伴互面和面后反馈能补足纯 AI 模拟的临场压力。

当前产品已经把前四项接入到面试和记录主线里；coding 环境先通过实验室承接；真人/同伴压力模拟仍是后续缺口，短期可先用面试官 Agent 的自由讨论卡记录真人 mock 反馈。

### 面试官工作台当前能力

- 顶部独立工作区入口，和“面试者练习”并列。
- `mock-interviewer` Agent 负责：
  - 根据简历和可选 JD 生成整场题纲。
  - 按时长预算控制主问题数和追问数。
  - 在回答暴露职责不清、指标缺失、方案浅、取舍不明、岗位不对齐时动态追加追问。
  - 结束后给出总评、维度分、逐题聚合评分和参考好答案。
- 进行态改成双栏工作台：
  - 左栏一次展示全部主问题卡片。
  - 主问题下方可展开并重新聚焦编辑追问卡。
  - 左栏底部支持独立的自由讨论卡，不强挂原题。
  - 右栏是统一纪要区，支持对主问题、追问、自由讨论分别回填纪要。
- 完成态改成双层复盘：
  - 主问题复盘按“主问题纪要 + 所有关联追问纪要”聚合评分。
  - 自由讨论卡单独评分，不并入主问题。
  - 右侧展示总分、维度均分、优势和下一步动作。
- 历史恢复能力：
  - 历史面试记录会识别 `mock_interviewer` 会话并自动切到面试官工作区。
  - 完成后的结构化 summary 会写回 session，可重新进入查看，不只剩一句摘要。
- 使用连续性优化：
  - 切换主问题、追问卡和自由讨论卡时，会先缓存本地未提交纪要，回到该卡可继续编辑。
  - 最近面试列表会明确区分 `面试官 Agent` 和旧模拟面试，不再混在同一种卡片语义里。
  - 自由讨论卡可以手动归属到某个主问题，归属后会按追问并入该主问题复盘。

### 当前数据与接口

- `InterviewSession`
  - `contextJson`：保存 pasted resume、parsed resume、optional JD、role、seniority、duration。
  - `configJson`：保存 `sessionKind: "mock_interviewer"` 等产品配置。
  - `planJson`：保存题纲、主问题预算、追问预算、覆盖进度。
  - `expressionJson`：保存 agent 标识、优势、下一步动作，以及可恢复的结构化 `interviewerSummary`。
- `InterviewTurn`
  - `turnType`：`primary | followup | discussion`
  - `parentTurnId`：追问挂到主问题下，自由讨论为空
  - `intent`、`idealAnswer`、`reviewJson`
- Next API 与 `apps/api` 两套接口都已同步：
  - `POST /api/interviewer-sessions/start`
  - `POST /api/interviewer-sessions/:id/answer`
  - `POST /api/interviewer-sessions/:id/finish`

### 当前限制

- v1 仍然是文本纪要与文本评分，不做语音录入、实时转写和实时打分。
- 题纲暂不支持手动删题、改顺序、补题。
