# 项目功能演进记录

这份文档用于记录项目后续所有重要功能演进。以后每次做功能调整、模块收束、数据模型变化、主要 API 变化，都同步更新这里。

更新时间：2026-04-28

## 当前产品主线

项目现在收束为四个核心功能：

1. 记录
2. 面试
3. 技术文章
4. 创业想法

其他能力不再占主导航，而是作为这些核心功能里的辅助入口或弹窗，例如复盘、冲刺计划、趋势、公司情报、简历库、外部文章发现等。

## 2026-04-28：主功能收束

### 背景

之前 Web 端功能较多，顶部导航包含：首页、准备、八股、简历、模拟、实验室、公司、复盘、趋势等。功能虽然全，但对真实用户来说入口偏散，演示时也不够聚焦。

### 调整结果

主导航改为：

- 记录
- 面试
- 技术文章

旧能力保留，但下沉：

- 复盘：作为面试后的辅助入口。
- 冲刺计划：作为记录/复盘后的行动补充。
- 趋势：作为学习数据分析入口。
- 公司情报：作为记录和面试准备的补充入口。
- 简历库：作为面试功能内的辅助入口。

### 当前定位

Web 端继续作为主要产品调试台和真实功能验证入口。移动端仍然优先作为纯前端演示版，等 Web 端产品逻辑成熟后再逐步接入真实 API。

## 2026-04-28：记录功能

### 目标

用户看到好的八股文、知识点、文章摘录时，可以直接粘贴保存。保存后能在同一个页面搜索、查看原文、回顾和标记掌握。

### 已完成

- 新增“记录”作为主入口。
- 支持快速录入标题、主题、标签、正文。
- 保存后进入现有知识卡体系。
- 支持搜索已保存内容。
- 支持查看原文详情。
- 支持“已回顾”“还不熟”等回顾动作。
- 支持从记录页进入复盘、冲刺、趋势、公司情报等辅助能力。

### 当前实现

记录功能复用了现有 `KnowledgeCard` 数据模型。它适合保存八股、知识点、摘录和可回顾内容。

## 2026-04-28：面试功能

### 目标

面试功能聚合“面试官”和“面试者”两个视角，用户可以随时切换：

- 面试官：贴简历，选择面试方向、难度、题数，生成文字版面试文稿。
- 面试者：基于文稿逐题练习，也可以启动 AI 文字模拟面试。

### 已完成

- “面试”成为主入口。
- 增加“面试官文稿 / 面试者练习”双模式切换。
- 新增文字版面试文稿生成接口。
- 面试官文稿包含：
  - 开场提示
  - 主问题
  - 考察意图
  - 追问
  - 优秀信号
  - 风险信号
  - 评分口径
  - 收尾话术
- 面试者练习支持：
  - 按文稿逐题回答
  - 暂存回答
  - 上一题/下一题切换
  - 启动 AI 文字模拟
- 最近面试记录保留在面试页底部。

### 新增接口

```text
POST /api/interview-script
```

该接口根据简历、岗位、方向、难度、题数生成文字版面试文稿。

## 2026-04-28：技术文章独立知识库

### 目标

技术文章不再只是外部文章推荐列表，而是独立的本地文章库。用户可以新建文章、填写主题页签和标签、粘贴文字和图片，并长期保存。

### 产品参考

- Notion：页面/数据库式保存和编辑。
- Raindrop：collection、tag、search 的知识管理方式。
- Readwise Reader：阅读、保存、回顾的知识流。

### 已完成

- “技术文章”保留为主入口。
- 技术文章改为本地文章库。
- 支持动态主题页签。
- 支持搜索标题、正文、摘要、标签。
- 支持标签筛选。
- 支持新建、编辑、删除文章。
- 支持文章详情展示。
- 支持来源 URL。
- 支持富文本正文。
- 支持图片粘贴/上传。
- 支持外部文章发现弹窗。
- 外部推荐文章可以保存为本地草稿。

### 数据模型

新增独立数据表：

```text
TechnicalArticle
TechnicalArticleAsset
```

`TechnicalArticle` 保存：

- 标题
- 主题 topic
- 标签 tagsJson
- 富文本 HTML
- 纯文本 contentText
- 来源 URL
- 摘要
- 状态
- 创建/更新时间

`TechnicalArticleAsset` 保存：

- 文章关联 ID
- 图片 URL
- 本地文件路径
- MIME 类型
- 文件大小
- alt 文本
- 创建时间

图片文件保存到：

```text
public/uploads/articles/
```

该目录已加入 `.gitignore`，避免把用户粘贴图片提交进仓库。

### 新增接口

```text
GET    /api/articles
POST   /api/articles
GET    /api/articles/[id]
PATCH  /api/articles/[id]
DELETE /api/articles/[id]
POST   /api/articles/assets
GET    /api/articles/discover
```

说明：

- `/api/articles` 现在服务本地技术文章库。
- `/api/articles/discover` 保留原来的外部文章推荐能力。
- `/api/articles/assets` 只接受 `png/jpeg/webp/gif` 图片，单图限制 10MB。

### 编辑器

已接入 TipTap，支持：

- 标题
- 段落
- 加粗
- 斜体
- 列表
- 引用
- 代码块
- 链接
- 图片
- 粘贴图片自动上传并插入正文

### 当前限制

- v1 只保存和展示图片，不做 OCR。
- 本地图片暂存在 `public/uploads/articles/`。
- 删除文章会删除数据库里的图片记录，本地文件 v1 不强制清理。

## 2026-04-28：后五项能力闭环

在主功能收束之前，Web 端补齐了一批辅助能力，这些能力现在作为补充入口保留：

- 复盘卡任务化：复盘卡可改状态、标记完成、转冲刺任务。
- 冲刺计划详情：支持选中计划、查看进度、按 Day 推进任务。
- 面经/公司情报增强：支持面经筛选、角色快捷筛选、公司情报下一步动作。
- 趋势页：支持准备度、面试分数走势、知识掌握分布、冲刺完成进度。
- 数据导出：支持 JSON 数据导出。

### 新增接口

```text
PATCH /api/reviews/[id]
GET   /api/export
```

## 2026-04-28：创业想法记录页

### 目标

新增一个独立的“创业想法”页，用来记录产品 idea、用户痛点、目标用户、解决方案、AI Agent 工作流、数据输入、商业化、验证计划和风险。第一条内置想法是“AI 播客生成 Agent”：根据用户的听歌习惯、播客收听历史和兴趣画像，由 AI Agent 生成一档个性化播客。

### 已完成

- 顶部主导航新增“创业想法”。
- 新增创业想法列表、搜索、状态筛选、标签筛选。
- 支持新建、编辑、删除创业想法。
- 支持“AI 播客模板”一键带入结构化内容。
- 已将“AI 播客生成 Agent”作为第一条本地记录保存。
- 想法详情按模块展示：
  - 用户痛点
  - 目标用户
  - 解决方案
  - 数据输入
  - AI Agent 工作流
  - 商业化
  - 验证计划
  - 风险

### 数据模型

新增独立数据表：

```text
StartupIdea
```

`StartupIdea` 保存：

- 标题
- 一句话描述 oneLiner
- 用户痛点 problem
- 目标用户 targetUsers
- 解决方案 solution
- AI Agent 工作流 aiAgentFlow
- 数据输入 dataSignals
- 商业化 monetization
- 验证计划 validationPlan
- 风险 risks
- 标签 tagsJson
- 状态 status
- 创建/更新时间

### 新增接口

```text
GET    /api/startup-ideas
POST   /api/startup-ideas
GET    /api/startup-ideas/[id]
PATCH  /api/startup-ideas/[id]
DELETE /api/startup-ideas/[id]
```

### 当前定位

创业想法页先作为产品孵化和演示入口，不直接混入“记录”或“技术文章”表。后续如果要把某个 idea 进入正式验证阶段，可以再扩展成脚本生成、MVP 任务拆解、用户访谈记录和验证数据看板。

## 2026-04-28：前五项能力补齐

Web 端此前补齐了第一批基础闭环：

- 首页/今日任务：聚合待复习、复盘卡、冲刺任务、最近面试。
- 题库复习闭环：支持编辑、删除、复习模式、掌握度更新。
- 简历管理：支持多简历、重命名、删除、跳转 JD 匹配。
- JD 匹配报告：展示匹配分、岗位职责、面试重点、优势、缺口、项目讲述路线。
- 模拟面试历史详情：支持查看历史面试、轮次详情、复盘报告。

### 新增接口

```text
PATCH  /api/knowledge/[id]
DELETE /api/knowledge/[id]
PATCH  /api/resumes/[id]
DELETE /api/resumes/[id]
```

## 2026-04-28：移动端定位调整

移动端当前不作为真实数据主入口，而是作为演示版：

- Expo SDK 已升级到 55。
- 移动端先做纯前端演示。
- 不依赖后端、不依赖登录、不依赖数据库。
- 主要用于给别人展示产品完整形态。

长期目标仍然是：

```text
Expo React Native App + NestJS API + Supabase Auth/Postgres + shared types
```

但短期优先在 Web 端打磨真实产品逻辑。
